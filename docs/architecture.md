# Architecture

## Overview

Billable is a local-first Windows desktop application for tracking consulting work, notes, categories, and invoicing metadata.

- Platform: Windows
- Runtime: .NET 8
- UI: WPF (MVVM)
- Database: SQLite (file-based, single-user)
- Markdown: Markdig (or similar)
- Attachments: File paths or BLOBs (configurable)

Internal namespaces are domain-driven and brand-agnostic:

- WorkTracking.Core
- WorkTracking.Data
- WorkTracking.UI

The app is designed for a single consultant, running locally, with no server dependency.

Planned solution layout (to be generated by Copilot):

## MVVM pattern

- **Model**: Plain C# classes from `WorkTracking.Core`.
- **ViewModel**: Exposes observable properties, commands, and collections for the UI.
- **View**: WPF XAML, bound to ViewModels via DataContext.

Key ViewModels:

- `MainWindowViewModel`
- `ClientListViewModel`
- `ClientDetailViewModel`
  - Tabs: Timesheet, Invoices, Summary, Settings
- `TimesheetViewModel`
- `InvoicePrepViewModel`
- `GlobalReportsViewModel`

---

## Data access

- Library: `Microsoft.Data.Sqlite` (or similar)
- Pattern: Repository + lightweight query helpers
- Connection: Single SQLite file (e.g., `billable.db`) in user data folder
- Migrations: On startup, ensure schema exists and apply any upgrades

Repositories (examples):

- `IClientRepository`
- `IWorkEntryRepository`
- `IWorkCategoryRepository`
- `IInvoiceRepository`
- `IAttachmentRepository`
- `ISettingRepository`

---

## Date and time handling

- All dates stored as ISO-8601 TEXT in SQLite:
  - `YYYY-MM-DD` or `YYYY-MM-DDTHH:MM:SS`
- Use SQLite `strftime` for grouping and filtering.
- In C#, use `DateTime` / `DateOnly` and convert to/from string at the data layer.

---

## Markdown and attachments

- Markdown stored as TEXT in `work_entry.notes_markdown`.
- Rendering:
  - Use Markdig to convert Markdown to HTML.
  - Display via WPF WebBrowser or a Markdown-capable control.
- Attachments:
  - Either:
    - Store file paths in DB and keep files on disk, or
    - Store BLOBs in DB.
  - Initial implementation: file paths (simpler, more transparent).

---

## Dependency injection

Use `Microsoft.Extensions.DependencyInjection`:

- Register repositories
- Register services (navigation, dialogs, markdown)
- Register ViewModels
- Configure root `MainWindow` with DI

---

## Error handling and logging

- Logging: `Microsoft.Extensions.Logging`
- Log to:
  - Console (debug)
  - Rolling log file in app data folder
- Handle:
  - DB connection issues
  - File IO errors
  - Validation errors
- Show user-friendly messages via dialog service.

---

## Non-functional goals

- Local-first, offline-capable
- Fast startup
- Simple deployment (single EXE or MSIX)
- No external services required
- Easy backup (single DB file + attachments folder)
